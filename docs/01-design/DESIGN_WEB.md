# Web UI Design

## Overview

The web UI is a **secondary interface** to the CLI. It provides:
- Rich chat interface with streaming responses
- Visual task browsing and management
- Attachment previews
- Configuration management

**Design Philosophy:** Simple, fast, server-rendered (HTMX), minimal JavaScript

## Technology Stack

- **Framework:** HTMX + Jinja2 templates
- **Styling:** Tailwind CSS (for rapid, modern UI)
- **Real-time:** WebSocket for chat streaming
- **Icons:** SVG icons (no external dependencies)

## Component Library

### Message Bubble

```html
<div class="message {role}">
  <div class="avatar">{icon}</div>
  <div class="content">
    <div class="text">{message}</div>
    <div class="timestamp">{time}</div>
  </div>
</div>
```

**Styles:**
- User: Right-aligned, blue background
- AI: Left-aligned, gray background
- System: Centered, yellow background (for notifications)

### Action Button

```html
<button class="btn-action"
        hx-post="/api/tasks/{id}/progress"
        hx-target="#task-{id}"
        hx-swap="innerHTML">
  Mark In Progress
</button>
```

### Task Card

```html
<div class="task-card" id="task-{id}">
```

## Color Scheme

### Status Colors
- **Pending:** Gray (`bg-gray-100`, `text-gray-700`)
- **In Progress:** Green (`bg-green-100`, `text-green-700`)
- **Completed:** Blue (`bg-blue-100`, `text-blue-700`)

### Priority Colors
- **Low:** Gray (`bg-gray-50`, `text-gray-600`)
- **Medium:** Yellow (`bg-yellow-50`, `text-yellow-600`)
- **High:** Orange (`bg-orange-50`, `text-orange-600`)
- **Critical:** Red (`bg-red-50`, `text-red-600`)

### Accents
- **Primary:** Indigo (`indigo-600`)
- **Secondary:** Slate (`slate-700`)
- **Success:** Green (`green-600`)
- **Warning:** Yellow (`yellow-600`)
- **Error:** Red (`red-600`)

---

## Responsive Design

### Mobile (< 768px)
- Single column layout
- Hamburger menu for settings
- Stacked task cards
- Simplified chat (hide suggested actions)

### Tablet (768px - 1024px)
- Two column where applicable
- Medium task cards
- Full chat interface

### Desktop (> 1024px)
- Multi-column where appropriate
- Large task cards
- Rich chat with side panel for context

## HTMX Interactions

### Task List Filter
```html
<div id="task-list">
  <form hx-get="/api/tasks" hx-target="#task-list">
    <select name="status">
      <option value="all">All</option>
      <option value="pending">Pending</option>
    </select>
    <input type="search" name="q" placeholder="Search...">
  </form>
  {task_cards}
</div>
```

### Real-time Chat Streaming
```html
<div id="chat-messages"
     hx-ext="sse"
     sse-connect="/api/chat/stream"
     sse-swap="innerHTML">
  {messages}
</div>
```

### Dynamic Task Update
```html
<button hx-post="/api/tasks/abc123/progress"
        hx-indicator="#loading-abc123"
        hx-swap="outerHTML"
        hx-target="#task-abc123">
  Start
</button>

<div id="loading-abc123" class="htmx-indicator hidden">
  <span>Updating...</span>
</div>
```

- Lazy loading for attachments
- Infinite scroll for task lists (page 10 tasks)
- Debounced search input (300ms)
- Image optimization for attachment previews
- Minimize JavaScript bundle size
